# User Stories - AqStream

## Обзор
Пользовательские истории для всех ролей в системе AqStream, сгруппированные по персонам и приоритетам.

## Персоны

### 1. Гость (Незарегистрированный пользователь)
Человек, который зашел на платформу без регистрации.

### 2. Участник (Зарегистрированный пользователь)
Человек, который ищет и посещает события.

### 3. Организатор
Человек или представитель организации, который создает и управляет событиями.

### 4. Администратор
Сотрудник платформы, который модерирует контент и управляет системой.

## User Stories по ролям

### Гость

#### Просмотр событий
```
Как Гость,
Я хочу просматривать список всех публичных событий,
Чтобы понять, какие мероприятия доступны на платформе.

Критерии приемки:
- Вижу карточки событий с основной информацией
- Могу открыть детальную страницу события
- Вижу кнопку "Войти для регистрации" вместо "Записаться"
```

#### Поиск событий
```
Как Гость,
Я хочу искать и фильтровать события,
Чтобы найти интересующие меня мероприятия.

Критерии приемки:
- Доступен поиск по ключевым словам
- Работают фильтры: дата, локация, тип, цена
- Результаты обновляются динамически
```

#### Регистрация
```
Как Гость,
Я хочу зарегистрироваться на платформе,
Чтобы записываться на события.

Критерии приемки:
- Регистрация через email/телефон
- Подтверждение через Telegram (опционально)
- Автоматический вход после регистрации
```

### Участник (Пользователь)

#### Запись на событие
```
Как Участник,
Я хочу записаться на интересующее меня событие,
Чтобы принять в нем участие.

Критерии приемки:
- Кнопка "Записаться" на странице события
- Выбор типа регистрации (если доступны варианты)
- Подтверждение записи
- Событие появляется в "Моих событиях"
```

#### Оплата участия
```
Как Участник,
Я хочу оплатить участие в платном событии,
Чтобы подтвердить свое место.

Критерии приемки:
- Вижу QR-код для оплаты
- Могу загрузить подтверждение оплаты
- Вижу статус оплаты в личном кабинете
- Получаю уведомление о подтверждении
```

#### Управление записями
```
Как Участник,
Я хочу видеть все события, на которые я записан,
Чтобы планировать свое время.

Критерии приемки:
- Раздел "Мои события" в личном кабинете
- Фильтрация: предстоящие/прошедшие
- Возможность отменить участие
- Статус регистрации и оплаты
```

#### Участие в группах
```
Как Участник,
Я хочу видеть, в какую группу я распределен,
И указать свои предпочтения по группировке.

Критерии приемки:
- Вижу информацию о группах после распределения
- Могу оставить комментарий с предпочтениями
- Вижу других участников своей группы
```

#### Вступление в сообщество
```
Как Участник,
Я хочу вступить в закрытое сообщество,
Чтобы видеть корпоративные или специальные события.

Критерии приемки:
- Могу отправить запрос на вступление
- Вижу статус запроса
- После одобрения вижу закрытые события
- Фильтр по сообществам в списке событий
```

#### Настройка уведомлений
```
Как Участник,
Я хочу получать уведомления о моих событиях,
Чтобы не пропустить важную информацию.

Критерии приемки:
- Привязка Telegram аккаунта
- Выбор типов уведомлений
- Уведомления в личном кабинете
- Push через Telegram bot
```

#### Лист ожидания
```
Как Участник,
Я хочу встать в лист ожидания на заполненное событие,
Чтобы получить место при появлении.

Критерии приемки:
- Кнопка "Встать в лист ожидания"
- Вижу свою позицию в очереди
- Получаю уведомление при освобождении места
```

### Организатор

#### Создание события
```
Как Организатор,
Я хочу создать новое событие,
Чтобы привлечь участников.

Критерии приемки:
- Форма с полями: название, описание, дата, место, лимит
- Выбор типа события
- Настройка типов регистрации
- Сохранение как черновик
- Отправка на модерацию
```

#### Управление регистрациями
```
Как Организатор,
Я хочу управлять записями участников,
Чтобы контролировать состав мероприятия.

Критерии приемки:
- Список всех регистраций
- Подтверждение/отклонение записей
- Массовые действия
- Экспорт списка участников
```

#### Подтверждение оплаты
```
Как Организатор,
Я хочу подтверждать оплату от участников,
Чтобы фиксировать финансовые поступления.

Критерии приемки:
- Список ожидающих подтверждения
- Просмотр загруженных чеков
- Массовое подтверждение
- Отметка о возврате при отмене
```

#### Формирование групп
```
Как Организатор,
Я хочу распределить участников по группам,
Чтобы организовать логистику мероприятия.

Критерии приемки:
- Выбор шаблона группировки
- Настройка параметров групп
- Ручное распределение участников
- Учет предпочтений участников
- Публикация результатов
```

#### Управление сообществом
```
Как Организатор,
Я хочу создать закрытое сообщество,
Чтобы проводить корпоративные мероприятия.

Критерии приемки:
- Создание сообщества
- Одобрение запросов на вступление
- Управление участниками
- Создание закрытых событий
```

#### Отмена события
```
Как Организатор,
Я хочу иметь возможность отменить событие,
Если возникли непредвиденные обстоятельства.

Критерии приемки:
- Кнопка отмены с подтверждением
- Автоматические уведомления участникам
- Список для возврата средств
- Изменение статуса на "Отменено"
```

#### Дублирование события
```
Как Организатор,
Я хочу дублировать существующее событие,
Чтобы быстро создавать повторяющиеся мероприятия.

Критерии приемки:
- Кнопка "Дублировать"
- Копирование всех настроек
- Возможность изменить дату и детали
- Сохранение как нового события
```

#### Просмотр аналитики
```
Как Организатор,
Я хочу видеть статистику по моим событиям,
Чтобы понимать эффективность и планировать улучшения.

Критерии приемки:
- Дашборд с ключевыми метриками
- Статистика по типам событий
- Динамика регистраций
- Конверсия просмотров в регистрации
- Финансовые показатели
```

#### Шаблоны событий
```
Как Организатор,
Я хочу сохранять события как шаблоны,
Чтобы быстро создавать типовые мероприятия.

Критерии приемки:
- Сохранение события как шаблон
- Библиотека шаблонов
- Создание события из шаблона
- Редактирование шаблонов
```

### Администратор

#### Модерация событий
```
Как Администратор,
Я хочу модерировать новые и измененные события,
Чтобы обеспечить качество контента на платформе.

Критерии приемки:
- Очередь на модерацию
- Просмотр всех деталей события
- Одобрение/отклонение с комментарием
- Установка автоодобрения для организатора
```

#### Управление пользователями
```
Как Администратор,
Я хочу управлять пользователями системы,
Чтобы поддерживать порядок на платформе.

Критерии приемки:
- Список всех пользователей
- Поиск и фильтрация
- Назначение ролей
- Блокировка/разблокировка
- История действий пользователя
```

#### Управление организациями
```
Как Администратор,
Я хочу создавать и управлять организациями,
Чтобы структурировать организаторов.

Критерии приемки:
- Создание организации
- Привязка организаторов
- Установка лимитов и правил
- Мониторинг активности
```

#### Системные настройки
```
Как Администратор,
Я хочу управлять настройками платформы,
Чтобы адаптировать систему под потребности.

Критерии приемки:
- Управление типами событий
- Настройка шаблонов групп
- Конфигурация уведомлений
- Настройка модерации
```

#### Мониторинг платформы
```
Как Администратор,
Я хочу видеть общую статистику платформы,
Чтобы понимать состояние и развитие сервиса.

Критерии приемки:
- Общее количество пользователей/событий
- Активность по периодам
- Популярные категории
- Проблемные области
- Системные метрики
```

## Приоритизация

### Must Have (MVP)
1. Регистрация и вход
2. Создание события (Организатор)
3. Просмотр событий (Все)
4. Запись на событие (Участник)
5. Управление регистрациями (Организатор)
6. Модерация событий (Администратор)

### Should Have (Фаза 2)
1. Поиск и фильтрация
2. Telegram уведомления
3. Оплата и подтверждение
4. Базовые группы
5. Сообщества
6. Дашборд организатора

### Could Have (Фаза 3)
1. Лист ожидания
2. Расширенные шаблоны групп
3. Дублирование событий
4. Шаблоны событий
5. Расширенная аналитика

### Won't Have (в MVP)
1. Email уведомления
2. Мобильные приложения
3. Интеграция с платежными системами
4. API для третьих сторон
5. Telegram Mini App

## Метрики успеха историй

Для каждой истории определяем:
- **Время выполнения**: целевое время для завершения пользователем
- **Успешность**: % успешных завершений с первой попытки
- **Удовлетворенность**: оценка пользователей (1-5)

Целевые показатели:
- Критические пути < 2 минут
- Успешность > 90%
- Удовлетворенность > 4.0
