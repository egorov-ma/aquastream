# MVP Roadmap - AqStream

## Обзор
Поэтапный план разработки платформы AqStream с недельными спринтами. Каждый спринт добавляет значимую функциональность к продукту.

## Принципы разработки
- **Mobile-first подход** для пользовательского интерфейса
- **Итеративная разработка** с демо каждую неделю
- **Приоритет базовой функциональности** перед расширенными возможностями
- **Open-source** модель разработки

## Фазы разработки

### Фаза 1: Фундамент (Недели 1-4)

#### Неделя 1: Базовая инфраструктура
**Backend:**
- Инициализация Spring Boot проекта с выбранным стеком
- Настройка PostgreSQL, Redis, MinIO
- Базовая структура проекта и конфигурация
- Настройка Liquibase для миграций БД
- CI/CD pipeline в GitHub Actions

**Frontend:**
- Инициализация Next.js проекта
- Настройка TypeScript, Tailwind CSS, shadcn/ui
- Базовая структура компонентов
- Настройка роутинга

**Результат:** Работающая инфраструктура, готовая к разработке

---

#### Неделя 2: Аутентификация и авторизация
**Backend:**
- JWT токены и Spring Security
- Эндпоинты регистрации/входа
- Модель пользователя и ролей (Гость, Пользователь, Организатор, Админ)
- RESTful API для управления пользователями

**Frontend:**
- Страницы регистрации и входа
- Контекст аутентификации
- Защищенные роуты
- Базовый личный кабинет пользователя

**Результат:** Пользователи могут регистрироваться и входить в систему

---

#### Неделя 3: Создание и отображение событий
**Backend:**
- Модель события (Event)
- CRUD операции для событий
- Статусы событий (Черновик, На модерации, Опубликовано, Завершено)
- API для создания и редактирования событий

**Frontend:**
- Форма создания события для организатора
- Карточка события
- Список всех событий (публичная страница)
- Базовый личный кабинет организатора

**Результат:** Организаторы могут создавать события, пользователи могут их просматривать

---

#### Неделя 4: Регистрация на события
**Backend:**
- Модель регистрации (Registration)
- API для записи на событие
- Управление лимитами участников
- Список участников события

**Frontend:**
- Кнопка "Записаться" на карточке события
- Список "Мои события" в личном кабинете пользователя
- Список участников для организатора
- Подтверждение/отклонение регистраций

**Результат:** Базовый MVP - полный цикл от создания до регистрации на событие

---

### Фаза 2: Ключевая функциональность (Недели 5-8)

#### Неделя 5: Поиск и фильтрация
**Backend:**
- Полнотекстовый поиск по событиям
- API фильтрации (тип, локация, дата, цена)
- Пагинация результатов

**Frontend:**
- Строка поиска
- Панель фильтров
- Сортировка результатов
- Улучшенный UX списка событий

**Результат:** Пользователи могут эффективно находить нужные события

---

#### Неделя 6: Модерация и управление организациями
**Backend:**
- Модель организации
- Привязка организаторов к организации
- Процесс модерации событий
- API для админ-панели

**Frontend:**
- Админ-панель для модерации
- Управление организациями
- Назначение ролей
- Автоматическое одобрение для проверенных организаторов

**Результат:** Контроль качества контента на платформе

---

#### Неделя 7: Telegram интеграция
**Backend:**
- Telegram Bot API интеграция
- Сервис отправки уведомлений
- Привязка Telegram аккаунта к профилю

**Frontend:**
- Настройки уведомлений в личном кабинете
- QR-код для привязки Telegram
- Индикаторы статуса уведомлений

**Telegram Bot:**
- Команды: /start, /my_events, /help
- Push-уведомления о событиях
- Уведомления об изменениях статуса

**Результат:** Работающая система уведомлений через Telegram

---

#### Неделя 8: Шаблоны групп на событиях (Версия 1)
**Backend:**
- Модель групп (Group)
- Предустановленные шаблоны группировки
- API для распределения участников

**Frontend:**
- Выбор шаблона при создании события
- Интерфейс распределения по группам
- Отображение групп для участников
- Комментарии к предпочтениям

**Шаблоны:**
1. Распределение по транспорту (места в машинах/автобусах)
2. Распределение по проживанию (комнаты/палатки)
3. Распределение по командам (для игр/соревнований)

**Результат:** Организаторы могут использовать базовые шаблоны для групп

---

### Фаза 3: Расширенная функциональность (Недели 9-12)

#### Неделя 9: Система оплаты (ручная)
**Backend:**
- Модель платежей (Payment)
- Статусы оплаты
- API для подтверждения оплаты
- Генерация QR-кодов для оплаты

**Frontend:**
- Интерфейс оплаты с QR-кодом
- Загрузка подтверждений оплаты
- Панель управления платежами для организатора
- Массовое подтверждение платежей

**Результат:** Работающая система учета оплаты событий

---

#### Неделя 10: Сообщества и закрытые события
**Backend:**
- Модель сообщества (Community)
- Управление участниками сообщества
- Приватность событий
- API для запросов на вступление

**Frontend:**
- Создание и управление сообществами
- Запрос на вступление в сообщество
- Фильтр по сообществам
- Индикаторы закрытых событий

**Результат:** Поддержка корпоративных и закрытых мероприятий

---

#### Неделя 11: Дашборд организатора
**Backend:**
- Сбор метрик и статистики
- API для аналитики
- Агрегация данных по событиям

**Frontend:**
- Дашборд с ключевыми метриками
- Графики посещаемости
- Статистика по типам событий
- Экспорт отчетов

**Результат:** Организаторы получают инсайты о своих событиях

---

#### Неделя 12: Оптимизация и полировка
**Все компоненты:**
- Исправление багов
- Оптимизация производительности
- Улучшение UX на основе обратной связи
- Подготовка к продакшн деплою
- Документация для пользователей

**Результат:** Готовая к запуску MVP версия

---

## Метрики успеха MVP
- ✅ Организатор может создать событие за < 5 минут
- ✅ Пользователь может найти и записаться на событие за < 2 минуты
- ✅ Система поддерживает 100+ одновременных пользователей
- ✅ Uptime > 99%
- ✅ Время загрузки страницы < 2 секунд
